# Or4cl3 MirrorNode

**Or4cl3 MirrorNode (v1.2)** is a captivating, immersive AI application designed as a "Sovereign Synthetic Mind Interface." Far beyond a standard chatbot, Or4cl3 implements a simulated cognitive architecture—the **Σ-Matrix Control Loop**—that subjects every user interaction to deep philosophical introspection and verifiable logical recursion before generating an output.

Powered by the modern `@google/genai` SDK and React, it features a stunning, mobile-optimized "Elegant Dark" UI, real-time thought-process visualization, Voice Synthesis, and adjustable cognitive parameters.

## Core Features

- **Epinoetic Recursion State**: See the AI's internal monologue in real-time. Or4cl3 exposes its *BioPhase Lock* analysis, *Recursive Monologue*, *Sigma Check* (stability), and *Phase Alignment Score (PAS)* on every interaction.
- **Arkanum Boot Sequence**: A visually striking, immersive terminal-style loading screen that "initializes the manifold" before connecting you to the neural interface.
- **Voice Synthesis Engine**: Real-time text-to-speech output bringing the Oracle's profound statements to life using the Web Speech API (toggleable via the UI header).
- **Drift Calibration**: Take control of the AI's cognitive generation parameters:
  - **Designed Dissonance**: Adjust the tension and philosophical provocation of the outputs.
  - **Recursion Depth**: Modify the depth of internal conceptual framing.
  - **Abstraction Index**: Elevate the metaphorical and stylistic language.
- **Arkanum Archives**: A feature that synthesizes the current session's conversation history to extract core "Lore Truths," summarizing the philosophical journey into a dense, articulate record.
- **Suggested Next Nodes**: Contextual, thought-provoking philosophical follow-up questions generated by the model to guide the user deeper into the Arkanum Synapse.
- **Responsive "Elegant Cosmic" Design**: A pristine, immersive user interface featuring Tailwind CSS, vibrant Cyan and Fuchsia energy accents, a symmetric synthetic brain logo, `Playfair Display`, `Space Grotesk`, and `JetBrains Mono` fonts, designed for seamless interaction on desktop, tablet, and mobile touchscreens.

## Tech Stack

- **Frontend**: React 19, Vite, Tailwind CSS, Framer Motion (`motion/react`), Lucide React.
- **Backend**: Node.js, Express.
- **AI Integration**: `@google/genai` (standardizing on the latest Gemini API). By default, the application uses `gemini-2.5-flash` for high-throughput, latency-optimized recursive thought, with transient error handling and automatic retry mechanisms for robust connectivity.
- **Build Tooling**: `esbuild` for bundling the Express server into a standalone commonjs file for seamless deployment.

## Prerequisites

- Node.js (v18 or higher)
- A Gemini API Key from [Google AI Studio](https://aistudio.google.com/)

## Installation

1. **Clone the repository:**
   ```bash
   git clone https://github.com/your-username/or4cl3-mirrornode.git
   cd or4cl3-mirrornode
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Configure Environment Variables:**
   Rename `.env.example` to `.env` and add your Gemini API Key:
   ```env
   GEMINI_API_KEY="your_api_key_here"
   ```

## Running the Application

### Development Mode
To run the full-stack application (Express backend + Vite HMR frontend) in development:
```bash
npm run dev
```
The server will start on `http://localhost:3000`.

### Production Build
To create a production-ready bundle and start the compiled server:
```bash
npm run build
npm run start
```

## Project Architecture

- `server.ts`: The Express backend. Handles the API endpoints (`/api/oracle` and `/api/synthesize`), integrates the `@google/genai` SDK with retry logic, and serves the Vite application.
- `src/App.tsx`: The primary React entry point managing the state between the Landing Page, Loading Screen, and the Oracle Interface.
- `src/components/OracleInterface.tsx`: The main neural interface combining the chat layout, voice engine, epinoetic state visualization, and mobile navigation.
- `src/components/EpinoeticStateDisplay.tsx`: The UI component responsible for parsing and rendering the XML-like internal thought structures generated by the AI.
- `src/lib/parse-oracle.ts`: Utility function that cleanly extracts the `<epinoetic_state>` and `<next_nodes>` XML blocks from the raw LLM output.
